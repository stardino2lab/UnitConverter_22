## §4 SRP / OCP 체크

| 모듈 | 단일 책임 | 하지 않는 일 |
|------|-----------|--------------|
| `length_unit.py` | 단위 계약 (`name`, `to_meter`) | 파싱, 출력, exit code |
| `unit_registry.py` | 이름 기준 등록·조회 | 변환 수식, CLI |
| `converter.py` | meter 허브 기반 전 단위 숫자 변환 | 문자열 포맷, 입력 검증 |
| `input_parser.py` | `단위:값` 파싱·형식·음수 검증 | 단위 존재 여부, 변환 연산 |
| `output_formatter.py` | README 형식 문자열 출력 | 변환 로직, 오류 판정 |
| `cli.py` | I/O 오케스트레이션·exit code | 개별 단위 `to_meter` 구현 |
| `config_loader.py` *(P1)* | JSON/YAML 비율 로드 | CLI 오케스트레이션 |

**OCP:** `inch` 등 추가 = `LengthUnit` 구현 + `register()` — `converter.py` 핵심 비수정.

**데이터 흐름 (P0 Happy Path):**

```
argv/stdin "meter:2.5"
  → input_parser.parse()        → ("meter", 2.5)
  → unit_registry.get("meter")
  → converter.convert_all(...)  → {feet, yard, meter, ...}
  → output_formatter.format(...) → stdout
```
